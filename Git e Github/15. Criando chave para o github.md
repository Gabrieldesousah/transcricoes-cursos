# Criando chave para o github

Agora mostraremos como você faz para gerar uma chave pública para o Github. Se você nunca ouviu falar, não se preocupe, nós explicaremos.

Toda vez que você criar um repositório no Github e for fazer um **push**, o Github irá pedir autenticação, mas como nós somos desenvolvedores e precisamos enviar estes arquivos o tempo todo, não seria legal ficar digitando toda vez, no terminal, seu login e senha para conseguir efetuar este push.

Falaremos mais sobre push em outros módulos, para explicar melhor este comando.

Então ao invés de você fazer isso nós criaremos uma chave de autenticação.

Para este processo nós iremos rodar um comando, em nosso terminal, que gerará dois arquivos.

1. Uma chave criptografada privada
2. Uma chave criptografada pública

Esta chave privada sempre vai ficar nos eu computador e você jamais irá compartilhar com ninguém, mas a pública você pode compartilhar com diversos serviços, inclusive o Github.

Você compartilhando esta chave com o Github, toda vez que você for executar qualquer operação no Github, o serviço irá verificar se a chave pública confere com a chave privada da nossa máquina, o Github irá verificar se realmente se trata de um par, caso ele identifique, a operação é liberada sem precisar de login e senha.

Esta autenticação acaba sendo muito mais segura do que trabalhar com usuário e senha.

## Gerando chaves no computador

Entre em seu terminal e digite o comando abaixo:

`$ ssh-keygen`

Este comando mostrará o caminho em que as chaves serão geradas e pede confirmação. Caso não exista nenhuma chave criada ele já criará automaticamente, caso já exista ele perguntará se quer sobrescrever.

Como você sabe se a chave está gerada e onde estão os arquivos? Você precisa acessar o caminho informado na criação e listar os arquivos internos. Veja o comando abaixo:

Primeiro `$ cd ~/.ssh/` depois `$ ls`.

Você deverá encontrar os arquivos:

* id_rsa
* id_rsa.pub

Percebam que o que tem a extensão **.pub** se trata da chave pública que iremos compartilhar com os serviços. Para pegar o conteúdo deste arquivo você pode rodar o seguinte comando:

`$ cat id_rsa.pub`

Depois copie o código para adicionar ao Github. Você pode também abrir o arquivo em algum bloco de notas se preferir, o importante é copiar o conteúdo deste arquivo.

Para cadastrar no Github vá até **Account Settings** e depois clique em **SSH Keys**. Neste local você vai perceber que existe a possibilidade de cadastrar diversas chaves. Crie uma nova chave. Você pode adicionar uma descrição para cada chave para saber de qual máquina ela pertence.

No campo da chave você irá colar o conteúdo copiado e clica em **add key**. Depois disso nós já teremos uma nova chave configurada e, a partir deste momento, todas as operações que forem efetuadas, do meu computador para o repositório online, não precisarão mais de autenticação por login e senha.
